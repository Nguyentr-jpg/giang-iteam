# Playbooks

Quy trình chi tiết cho các loại việc LẶP LẠI, để nhân viên làm nhất quán.

Giang/nhân viên chỉ đọc file trong đây khi task khớp loại việc tương ứng —
không đọc hết mỗi lần chạy (tránh tốn context).

Chưa có playbook nào. Ví dụ sẽ thêm về sau:
- `supabase-migration.md` — quy trình viết migration idempotent an toàn.
- `new-endpoint.md` — chuẩn thêm endpoint mới.
- `blog-pipeline.md` — quy trình chạy pipeline bài viết (khi có nhân viên content).

Thêm playbook = thêm 1 file, rồi tham chiếu từ SKILL.md của nhân viên liên quan.
